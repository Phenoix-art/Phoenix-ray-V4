What is G2Ray?

G2Ray runs Xray/VLESS inside GitHub Codespaces -- free proxy server with zero setup cost.

  • Fork the G2Ray repo on GitHub
  • Open Codespaces and let it auto-deploy
  • Grab your VLESS link from the terminal
  • Import into any V2Ray-compatible client
    • View on GitHub

    Supported Protocols

    All protocols run over port 443 with TLS. Choose the best fit for your censorship environment.

    VLESS + XHTTP

    Default transport. Fastest, lowest overhead. Works on most networks.

    VLESS + WebSocket

    Hides as regular WebSocket traffic. Good for heavily filtered networks.

    VLESS + HTTP/2

    Uses HTTP/2 multiplexing. Bypasses deep packet inspection on some ISPs.

    VLESS + gRPC

    gRPC transport. Resistant to DNS-level blocking.

    Fragment Mode

    Breaks TLS records into fragments. Evades SNI-based blocking.

    Noise (XUDP)

    UDP noise packets before TLS handshake. Masks traffic fingerprint.

    Setup Guides

    Step-by-step instructions for deploying your G2Ray proxy.

    Quick Start

    Deploy your G2Ray proxy in under 2 minutes using GitHub Codespaces.

    1. Go to github.com/edrisranjbar/g2ray and fork the repository
    2. Click the green Code button, then switch to the Codespaces tab
    3. Click Create codespace on main
    4. Wait 2-5 minutes for Xray to install
    5. Your VLESS link will print in the terminal

    Import into Your Client

    Copy the generated VLESS link and import it into your proxy client:

    V2RayNG (Android)

    Open app → Settings → Import from clipboard or scan QR code.

    Clash Meta

    Open Clash → Profiles → Add from clipboard or file.

    Nekoray (Desktop)

    Open Nekoray → Settings → Add new server from clipboard.

    Understanding Your Configuration

    The VLESS link follows this format:

    vless://UUID@HOST:PORT?encryption=none&type=NETWORK&security=TLS&sni=SNI#NAME

    Key parameters:

    • UUID -- Your unique identifier. Must be a valid v4 UUID.
    • HOST -- Server IP address or domain.
    • PORT -- Usually 443 for TLS.
    • type -- Network transport (xhttp, ws, h2, grpc).
    • security -- tls or reality.
    • sni -- Server Name Indication for TLS.

    Fragment Configuration

    Fragment breaks the TLS handshake into smaller pieces to evade SNI blocking.

    fragment=true&packetLength=100-200&interval=10-20

    Recommended values for different scenarios:

    # Iran (Shecan) -- use default fragment=100-200,10-20 # Heavy filtering -- try larger fragments fragment=500-700,15-25

    Network Recommendations

    Shecan (Free)

    Use ports 443 or 8443. XHTTP transport recommended.

    TciNet

    WebSocket with early data works best. Add fragment for reliability.

    International

    Any transport works. gRPC offers best stealth.

    Troubleshooting

    Codespace won't start ▼
    Check your GitHub Codespaces quota. Free tier gives 120 compute hours/month. Stop any running codespaces and try creating a new one.
    Connection times out ▼
    Try a different network transport (ws, h2, grpc). Change the server IP to a different datacenter. Enable fragment mode. Check if your ISP blocks the port.
    TLS certificate error ▼
    Verify your SNI matches the certificate. The GitHub Codespace URL follows the pattern: <codespace-name>-443.<region>.github.dev
    Slow speeds ▼
    Use xhttp-go mode for better throughput. Reduce fragment packet size. Try HTTP/2 transport. Check if another codespace is consuming bandwidth.
